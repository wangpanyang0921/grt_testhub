# 测试模板配置使用说明

## 概述

测试模板配置功能允许您通过前端页面管理测试用例生成模板，无需修改代码。系统会根据您的配置自动匹配需求文档内容，生成相应的测试点、测试场景、测试步骤和前置条件。

## 功能特点

- **可视化配置**：通过前端页面直观管理所有模板
- **智能匹配**：根据关键词自动匹配需求文档
- **优先级管理**：支持设置模板优先级，高优先级模板优先匹配
- **业务模块分类**：按业务模块组织模板，便于管理
- **一键初始化**：内置职业教育SaaS平台常用模板

## 使用步骤

### 1. 访问测试模板配置页面

1. 登录系统
2. 点击左侧菜单「配置中心」
3. 选择「测试模板配置」

### 2. 初始化默认模板（可选）

首次使用时，建议先初始化职业教育SaaS平台的默认模板：

1. 点击「初始化职业教育模板」按钮
2. 确认后系统将自动创建20+个常用模板
3. 包括课程管理、学员管理、教学管理、直播教学等模块

### 3. 创建新模板

#### 3.1 点击「新建模板」按钮

#### 3.2 填写模板信息

| 字段 | 说明 | 示例 |
|------|------|------|
| 模板名称 | 给模板起个名字 | 课程管理测试点 |
| 模板类型 | 选择模板用途 | 测试点模板/测试场景模板/前置条件模板 |
| 业务模块 | 所属业务模块 | 课程管理 |
| 匹配关键词 | 用逗号分隔的关键词 | 课程,课时,章节,课件 |
| 模板内容 | 具体内容 | 见下方说明 |
| 优先级 | 数字越小越优先 | 10（高优先级） |
| 启用状态 | 是否启用 | 启用 |

#### 3.3 不同类型模板的内容格式

**测试点模板**：
- 内容格式：列表形式，每行一个测试点
- 示例：
  ```
  课程发布流程验证
  章节顺序编排验证
  课时完成状态验证
  ```

**测试场景模板**：
- 内容格式：列表形式，每行一个场景
- 示例：
  ```
  学员选课报名场景
  课程学习进度记录场景
  课时完成状态变更场景
  ```

**前置条件模板**：
- 内容格式：文本，使用 `{module}` 作为模块变量
- 示例：
  ```
  1. 系统正常运行
  2. 教师账号已登录
  3. 课程分类已配置
  4. {module}模块可访问
  ```

**测试步骤模板**：
- 内容格式：JSON对象
- 示例：
  ```json
  {
    "step1": "登录系统",
    "step2": "进入课程管理页面",
    "step3": "点击新建课程按钮"
  }
  ```

#### 3.4 保存模板

点击「确定」按钮保存模板。

### 4. 管理模板

#### 4.1 查看模板列表

- 列表显示所有模板的基本信息
- 支持按类型、业务模块、启用状态筛选
- 支持按关键词搜索

#### 4.2 编辑模板

1. 在列表中找到要编辑的模板
2. 点击「编辑」按钮
3. 修改相关信息
4. 点击「确定」保存

#### 4.3 删除模板

1. 在列表中找到要删除的模板
2. 点击「删除」按钮
3. 确认删除

#### 4.4 启用/禁用模板

- 在列表中切换「启用状态」开关
- 禁用的模板不会参与匹配

### 5. 测试模板匹配

在正式使用前，可以测试模板是否能正确匹配：

1. 点击「测试匹配」按钮
2. 输入一段需求描述文本
3. 选择要测试的模板类型（可选）
4. 点击「测试匹配」
5. 查看匹配结果，确认哪些模板会被触发

## 工作原理

### 匹配逻辑

1. 系统分析需求文档/需求描述的内容
2. 遍历所有启用的模板
3. 检查需求内容是否包含模板的关键词
4. 如果包含，则将该模板的内容加入生成结果
5. 按优先级排序，高优先级模板优先

### 优先级说明

- 优先级范围：1-999
- 数字越小，优先级越高
- 相同优先级的模板按创建时间排序

### 关键词匹配规则

- 关键词用逗号分隔
- 只要需求内容包含任意一个关键词，就会匹配
- 关键词匹配是包含关系，不是精确匹配
- 示例：关键词"课程"可以匹配"课程管理"、"在线课程"等

## 最佳实践

### 1. 关键词设置

- **通用性**：设置通用的关键词，提高匹配率
- **准确性**：避免过于宽泛的关键词，防止误匹配
- **组合使用**：一个模板可以设置多个关键词，用逗号分隔

### 2. 优先级设置

- **核心功能**：设置为高优先级（1-50）
- **常用功能**：设置为中优先级（51-150）
- **边缘功能**：设置为低优先级（151-999）

### 3. 业务模块分类

- 按实际业务模块组织模板
- 便于管理和查找
- 支持自定义模块名称

### 4. 模板内容编写

- **测试点**：描述具体的验证项，使用"验证"、"检查"等词汇
- **测试场景**：描述业务流程，使用"场景"、"流程"等词汇
- **前置条件**：列出测试执行前必须满足的条件

## 常见问题

### Q1: 为什么我的模板没有匹配到？

可能原因：
1. 模板未启用
2. 关键词设置不准确
3. 需求内容不包含关键词
4. 优先级设置问题

解决方法：
- 检查模板启用状态
- 使用「测试匹配」功能验证
- 调整关键词设置

### Q2: 多个模板匹配到同一个需求怎么办？

系统会：
1. 收集所有匹配的模板内容
2. 按优先级排序
3. 合并所有内容到测试结果中

### Q3: 可以导入导出模板吗？

目前版本暂不支持导入导出功能，后续版本会考虑添加。

### Q4: 模板配置会影响已有的测试用例吗？

不会。模板配置只影响新生成的测试用例，不会影响已存在的测试用例。

### Q5: 如何备份模板配置？

建议：
1. 定期记录关键模板的配置信息
2. 使用版本控制工具管理配置变更
3. 后续版本会添加导出功能
