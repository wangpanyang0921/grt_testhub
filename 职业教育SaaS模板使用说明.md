# 职业教育 SaaS 平台测试模板使用说明

## 📋 概述

这套代码为职业教育 SaaS 平台添加了专门的测试模板，让系统能够：
1. **识别教育行业特有的业务关键词**（课程、学员、直播、考试等）
2. **生成针对性的测试点**（如"视频播放流畅性验证"、"考试防作弊验证"）
3. **输出符合教育场景的测试步骤**（如"教师登录→创建课程→编排章节→发布验证"）

---

## 📁 修改了哪些文件

### 1. `apps/requirement_analysis/advanced_analyzer.py`
**作用**：智能分析需求文档，识别测试类型和场景

**添加的内容**：
- ✅ 8大类教育业务测试点（课程/学员/教学/直播/证书/营销/内容/问答）
- ✅ 8大类教育业务测试场景（学习流程/教学流程/考试测评等）
- ✅ 6类教育专用测试类型（学习体验/教学功能/内容安全/防作弊等）

### 2. `apps/requirement_analysis/views.py`
**作用**：生成具体的测试用例内容

**添加的内容**：
- ✅ 8大类教育业务前置条件
- ✅ 8大类教育业务测试步骤

## 📝 代码结构详解（小白版）

### 1. 测试点代码结构

```python
# 【课程管理】相关测试点
if any(keyword in text for keyword in ["课程", "课时", "章节", "课件", "大纲"]):
    test_points.extend([
        "课程发布流程验证",
        "章节顺序编排验证",
        "课时完成状态验证",
        "课程上下架控制验证",
        "课程内容预览验证",
        "课程分类归属验证"
    ])
```

**解释**：
- `if any(keyword in text...)`：如果需求文档中包含这些关键词任意一个
- `test_points.extend([...])`：就添加这些测试点
- 你可以复制这段代码，修改关键词和测试点，添加自己的业务

### 2. 测试场景代码结构

```python
# 【学习流程场景】
if any(keyword in text for keyword in ["学习", "课程", "课时", "章节", "进度"]):
    scenarios.extend([
        "学员选课报名场景",
        "课程学习进度记录场景",
        "课时完成状态变更场景",
        "学习资料下载场景"
    ])
```

**解释**：
- 场景是更高层次的业务流程描述
- 测试点是具体的验证项
- 场景 + 测试点 = 完整的测试覆盖

### 3. 测试步骤代码结构

```python
# 【课程管理】测试步骤
if any(keyword in req_name for keyword in ["课程", "课时", "章节", "课件"]):
    if i == 1:
        steps.append(f"{i}. 教师登录系统，进入课程管理后台")
    elif i == 2:
        if scenario_key == 1:  # 正常路径
            steps.append(f"{i}. 点击创建课程，填写课程基本信息")
        elif scenario_key == 2:  # 异常路径
            steps.append(f"{i}. 尝试创建课程，输入异常数据")
```

**解释**：
- `i` 表示第几步（1、2、3、4）
- `scenario_key` 表示测试类型（1=正常路径，2=异常路径，3=边界值，4=性能，5=安全）
- 系统会根据需求名称匹配对应的步骤模板

## 🎨 如何添加自己的业务模板

### 示例：添加"实训管理"模块

**Step 1: 添加测试点**（在 `advanced_analyzer.py` 中）

找到 `_get_functional_test_points` 方法，在职业教育测试点代码块后面添加：

```python
        "实训过程跟踪验证",
        "实训成果提交验证",
        "实训成绩评定验证"
    ])
```

**Step 2: 添加测试场景**（在 `advanced_analyzer.py` 中）

找到 `_identify_test_scenarios` 方法，在职业教育场景代码块后面添加：

```python

if any(keyword in text for keyword in ["实训", "实践", "实验", "项目"]):
    scenarios.extend([
        "实训项目创建场景",
        "实训任务领取场景",
        "实训过程指导场景",
        "实训成果提交场景",
        "实训成绩评定场景"
    ])
```

**Step 3: 添加测试步骤**（在 `views.py` 中）

找到生成测试步骤的代码块，在问答互动步骤后面添加：

```python
# 【实训管理】测试步骤
elif any(keyword in req_name for keyword in ["实训", "实践", "实验", "项目"]):
    if i == 1:
        steps.append(f"{i}. 教师登录系统，创建实训项目")
    elif i == 2:
        if scenario_key == 1:
            steps.append(f"{i}. 分配实训任务给学员")
        elif scenario_key == 2:
            steps.append(f"{i}. 尝试分配异常任务（如过期任务）")
        else:

    elif i == 3:
        steps.append(f"{i}. 学员完成实训并提交成果")
    elif i == 4:
        steps.append(f"{i}. 教师评定成绩，验证实训记录")
```

**Step 4: 添加前置条件**（在 `views.py` 中）

找到生成前置条件的代码块，在问答互动前置条件后面添加：

```python

elif any(keyword in req_name for keyword in ["实训", "实践", "实验", "项目"]):
    precondition = f"1. 系统正常运行\n2. 教师账号已登录\n3. 实训环境已配置\n4. 学员名单已导入\n5. {req_module}模块可访问"
```

## ⚠️ 注意事项

1. **关键词匹配是"或"关系**：只要包含任意一个关键词就会触发
2. **代码缩进很重要**：Python 对缩进敏感，复制粘贴时要保持缩进一致
3. **重启服务生效**：修改代码后需要重启 Django 服务才能生效
4. **保留原有代码**：新增代码不要删除原有代码，用 `elif` 连接
5. **测试效果**：建议先用简单的需求测试，确认生效后再使用复杂需求

---

## 📞 遇到问题怎么办

1. **代码报错**：检查缩进是否正确，Python 必须用4个空格缩进
2. **不生效**：确认是否重启了服务，修改代码后必须重启
3. **匹配错误**：检查关键词是否写对，注意大小写敏感
4. **想恢复默认**：删除添加的代码块，保留原有的 `elif "登录"` 等代码
